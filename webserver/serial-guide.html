🔌 Serial-konsoll & Powerline-paring

📌 J6 — FTDI serial-konsoll

J6 er en 6-pinners 2.54 mm header på Curb-kortet, merket J6. Brukes til serial-konsoll når SSH ikke er tilgjengelig — f.eks. ved boot-problemer eller før nettverksoppsett. Spenningsnivå: 3.3V.

⚠ J6 har ingen VCC-pin — Curb strømsettes separat via vanlig strømkabel. Koble kun GND (pin 6), TX (pin 2) og RX (pin 3).

Serial-innstillinger

115200 baud  ·  8N1  ·  ingen flow control  ·  3.3V logikk

Login: root  /  curb123

🌐 Chrome Web Serial Monitor

Chrome har innebygget serial-terminal via Web Serial API — ingen installasjon nødvendig.

✓ Krever Chrome 89 eller nyere. Fungerer ikke i Firefox eller Safari.

Koble til

  1. Koble FTDI-adapteren til USB (GND + TX + RX som vist over). Curb kan ha strøm fra vanlig strømkabel.
  2. Åpne Chrome og gå til: 
    https://googlechromelabs.github.io/serial-terminal/
    Alternativt: Arduino Serial Monitor eller hvilken som helst side som bruker Web Serial API.
  3. Klikk Connect → velg FTDI-porten i listen (f.eks. USB Serial Device (COM3) på Windows).
  4. Sett baud rate til 115200, data bits 8, stop bits 1, parity None, flow control None.
  5. Klikk Connect igjen. Trykk Enter — du skal se login-prompt fra Curb.
  6. Logg inn: root → Enter → curb123 → Enter.

Tips

  • Reboot Curb mens terminalen er åpen for å se full boot-sekvens.
  • Trykk Enter et par ganger om ingen prompt vises — getty kan ha missett første karbonretur.
  • For å redusere støy i loggen: dmesg -n 1 etter innlogging.
  • For å skjule ekkoing av tastetrykkene: stty -echo

🔌 Powerline-paring (Curb ↔ HomePlug AV-adapter)

Manuell paring av Curb sin innebygde QCA7000 powerline-modem (eth1) mot en Curb-merket 600 Mbps HomePlug AV-adapter når midt-LED på adapteren er av (= ikke parret).

⚠ Adapter og Curb må være på samme elektriske krets. Begynn med samme stikkontakt for testing — kretsbrytere og faser kan blokkere powerline-signalet.

Du trenger

  • SSH eller serial-tilgang til Curb
  • Adapterens HWWC-kode (Device Password) — på baksiden, format: HWWC-XXXX-XXXX-XXXX
  • Adapterens MAC-adresse — også på label, format: XX:XX:XX:XX:XX:XX

Steg 1 — Finn Curb sin QCA7000 MAC

plcdevs

Output eksempel:

 eth1 00:08:ED:13:2C:45 192.168.050.050 QCA7000 MAC-QCA7000-1.1.0.730-...

Noter MAC-adressen (her: 00:08:ED:13:2C:45).

Steg 2 — Generer nøkler

# DAK fra adapterens HWWC-passord
hpavkey -D HWWC-XXXX-XXXX-XXXX

# NMK fra valgfri streng — velg noe du husker
hpavkey -M MITTNETTKEY123

Begge kommandoer gir 32 hex-tegn. Eksempel:

  • DAK: 48EF3AFBCFB72CAB467BB1FAB3D38114
  • NMK: 3581A85F9F0C734CBF8FD4D55F567AB7

Steg 3 — Sett NMK lokalt på Curb

int6k -i eth1 -K <NMK> -M <CURB_MAC>

Eksempel:

int6k -i eth1 -K 3581A85F9F0C734CBF8FD4D55F567AB7 -M 00:08:ED:13:2C:45

Forventet output: eth1 00:08:ED:13:2C:45 Set Local Network Membership Key

Steg 4 — Sett NMK remote på adapter (via DAK)

int6k -i eth1 \
  -D <DAK> \
  -K <NMK> \
  -J <ADAPTER_MAC> \
  <CURB_MAC>

Eksempel:

int6k -i eth1 \
  -D 48EF3AFBCFB72CAB467BB1FAB3D38114 \
  -K 3581A85F9F0C734CBF8FD4D55F567AB7 \
  -J 7C:8B:CA:AE:2A:AA \
  00:08:ED:13:2C:45

Forventet output: eth1 00:08:ED:13:2C:45 Set Remote Network Membership Key

Steg 5 — Verifiser

int6klist <CURB_MAC>

Skal vise begge MAC-er. Midt-LED på adapteren skal lyse.

Steg 6 — Hent IP via DHCP

udhcpc -i eth1

Feilsøking

Symptom Sannsynlig årsak
int6klist viser kun lokal MAC NMK ikke lik på begge sider — gjenta steg 3 og 4
Midt-LED av Som over, eller adapter på annen elektrisk krets
Sett-Remote feiler stille Feil DAK (sjekk HWWC-koden), eller adapter mangler strøm
Ingen DHCP-lease NMK OK men ingen DHCP-server på den andre siden
